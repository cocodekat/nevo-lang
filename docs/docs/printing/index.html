Printing

Released on 2025-10-11

Why do we have printing?

Printing is used in every language for debugging or terminal-app purposes. Let’s say we are building an app. And we have to check if our program is working. Instead of testing every single feature we can for example print when a function runs to check if that function actually ran, so it is very very popular.

How do we use it?

Now let’s try it out ourselves. Printing works almost the same as in c, just instead of printf we do print: 
print(“Hello world!“)

This will print hello world. We use the () as print is a function, Then we use ““ because we want to print a string. A string is just a piece of text. To print variables we (just like in c) do: 
print(“%variable_type“, variable)

So to print a int, we would do %d or %I, here are all of the types: 
Integer types
● %d or %i -> signed decimal integer (int)
● %u -> unsigned decimal integer (unsigned int)
● %o -> unsigned octal
● %x -> unsigned hexadecimal (lowercase a-f)
● %X -> unsigned hexadecimal (uppercase A-F)

Character and string
● %c -> single character (char)
● %s -> string (null-terminated char *)

Floating-point types
● %f -> decimal floating-point (e.g., 3.14)
● %e or %E -> scientific notation (e.g., 3.14e+00)
● %g or %G -> shortest representation, uses %f or %e
● %a or %A -> hexadecimal floating-point (C99)

Pointers
● %p -> pointer address (implementation-defined, usually hex)

Miscellaneous
● %% -> literal percent sign

Length modifiers
● h -> short (e.g., %hd for short int)
● l -> long (e.g., %ld for long int)
● ll -> long long (e.g., %lld)
● z -> size_t (e.g., %zu for unsigned size type)
● L -> long double (e.g., %Lf)
